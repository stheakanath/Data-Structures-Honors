What are Splay Trees?

 

Is another type of data structure invented by Daniel Dominic Sleator and Robert Endre Tarjan in 1985

 

Defined as a self adjusting binary search tree.

 

Also has another feature: Allows users to access elements that were recently accessed. Like cache!

 

Splay trees are also self optimizing.

 

Advantages

 

Self optimizing. Therefore it preforms similarly to a "Recent Files" feature.

It allows used nodes be more accessible

 

Very Efficient. Worst case is O(log n). And it uses very little memory.

 

Disadvantages

 

Height of the tree is linear. That's it!

 

Also splay trees can be changed just by reading it, modifying the whole tree.

 

Uses

 

Like a cache.

 

Another example to which a splay tree would be a good application would be network routers. Since routers get several packets from incoming IP addresses, it must send packets back to the receiver. In this case these IP addresses will be used again, therefore making it a good idea to keep the IP addresses closer to the root in order to access it quickly, giving it a fast performance.
